# `docs/` — Project Documentation

This directory holds **project-level documentation** that is *not* lesson content.

## Purpose

Use this folder for material that explains how the curriculum is built, maintained, and used — as opposed to the teaching material itself (which lives in `modules/`).

## What belongs here

- Educational philosophy and design rationale
- Authoring guides and the standard topic template specification
- Style guides (writing, notation, diagram conventions)
- The AI authoring workflow (ChatGPT / Claude Code / Gemini roles)
- Glossary of symbols and terminology used across modules
- Versioning, release notes, and roadmap detail

## What does NOT belong here

- Lessons, notebooks, or exercises → `modules/`
- Generated images, animations, or storyboards → `assets/`
- Capstone and student project scaffolding → `projects/`

> **Status:** Placeholder. Documentation will be added as the curriculum develops.
