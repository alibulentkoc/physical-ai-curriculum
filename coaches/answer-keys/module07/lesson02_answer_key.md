# Answer Key 1.2 — Path vs Trajectory: Geometry versus Timing

*Coaches' key. Multiple choice followed by model short answers and grading notes.*

---

**Q1 — B.** A path is geometry only: q(s), s in [0,1], no time.

**Q2 — B.** s is geometric progress; s=0.5 is the spatial midpoint, not the time midpoint.

**Q3 — A.** A trajectory composes a path with a time scaling: q(t)=q(s(t)).

**Q4 — False.** The same path admits many timings, so velocity profiles differ.

**Q5 — A.** Planning makes the path; time parameterization makes the trajectory.

---

**Q6 — model answer.** A path is the geometric curve the tool follows, q(s) with unitless s in [0,1] — no time. A trajectory is a path plus a time schedule, q(t)=q(s(t)), which has units of time and therefore velocity and acceleration. The trajectory carries time; the path does not.
*Grading: award credit for the core idea above; accept equivalent phrasings and valid examples.*

**Q7 — model answer.** Because the path fixes only geometry (q(s)); the timing s(t) is a free choice. Different s(t) — constant-rate, ease-in/out, fast-then-slow — produce different velocities and accelerations on the identical route, i.e. different trajectories.
*Grading: award credit for the core idea above; accept equivalent phrasings and valid examples.*

**Q8 — model answer.** s(t) rises from 0 with ṡ=0 at t=0, increases ṡ to a brisk value through the early/middle portion (steep s), then flattens (small ṡ) over the final stretch so progress crawls through the last 20%, reaching s=1 with ṡ=0 at t=T. It is a steep-then-shallow S that is flat at both ends.
*Grading: award credit for the core idea above; accept equivalent phrasings and valid examples.*

---

### Common misconceptions to watch for
- Treating s as time — reinforce s∈[0,1] is geometric progress.
- Believing the path fixes velocities — velocities come from s(t).
