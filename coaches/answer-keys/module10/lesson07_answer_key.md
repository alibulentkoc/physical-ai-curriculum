# Answer Key 2.3 — When the Mirror Drifts: Sync Error

**Q1 — B.** The twin and the reported state (→ 0 after a sync).
**Q2 — B.** The twin and reality's TRUE state (can remain after a perfect sync).
**Q3 — B.** Reality carries unmodeled effects it does not report.
**Q4 — B.** No — it can only mirror what reality reports.
**Q5 — True.** 'Synced to the report' ≠ 'matching reality.'

**Q6 — model answer.** Not necessarily. The divergence measures the gap to the *reported* state, which is zero — the sync is flawless. But if reality carries an unmodeled effect it didn't report (a hidden offset, an unsensed load), the twin still differs from the *true* state, and that gap doesn't appear in a report-based divergence. So: the twin matches the report exactly; whether it matches reality depends on what the report omits. This is the origin of the sim-to-real gap.
*Grading: require distinguishing report-gap (0) from true-gap (possibly nonzero).*
