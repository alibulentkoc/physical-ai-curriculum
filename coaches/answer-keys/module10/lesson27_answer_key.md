# Answer Key — Lesson 7.3

**MC1:** A — monitor → re-sync if drifted → predict → adapt.
**MC2:** A — re-syncing first would zero the divergence and erase the monitoring signal.
**MC3:** A — re-sync precedes deciding so predict/adapt run from the latest state, not a stale one.
**MC4:** A — the cycle introduces no new theory; it composes existing operators.
**TF5:** False — the twin advises; only the real robot acts.

**Short 6 — model answer.** Mirror (sync) and Simulate (run-ahead) underlie every stage; Monitor is the present-tense Reality↔Twin comparison ("what is happening now?"), Predict is the run-ahead forecast ("what happens next?"), and Adapt is pre-validate-and-choose — the three questions answered in turn each cycle. *Grading:* full credit for placing Mirror/Simulate as the substrate and mapping Monitor/Predict/Adapt to the three questions.
