# Answer Key 1.2 — Twin vs. Model vs. Simulation

**Q1 — B.** A description of how a system behaves.
**Q2 — B.** A model run forward to compute what would happen.
**Q3 — B.** The live pairing/synchronization to a specific real asset.
**Q4 — B.** They nest: model ⊂ simulation ⊂ twin.
**Q5 — True.** Every twin is a simulation; not every simulation is a twin.

**Q6 — model answer.** A simulation: it runs the Module 9 model forward (many times) but is not paired with or synced to any specific deployed robot — it describes a hypothetical/aggregate greenhouse, not *this* robot now. It contains a model and is a simulation, but lacks the live binding that makes a twin.
*Grading: require "simulation" + the missing-binding reason.*
