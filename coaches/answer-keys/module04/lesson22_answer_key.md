# Answer Key — Lesson 6.2: Reading a Robot into a Table

**Coaches only.** Formative.

1. **Building a DH table starts by** — numbering joints and identifying each joint's axis (z).
2. **Base axis perpendicular to next joint's axis gives** — α = ±90°.
3. **Link length appears in a (along x), not d** — True.
4. **3-DOF capstone arm (short).** a2 = 0.4 (upper-arm link); α1 = 90° (base swivel perpendicular to the arm plane).

**Challenge rubric.** Full credit: if every a = 0 but the arm has long links, the link lengths are missing from the table (they must appear as nonzero a, or d for axial offsets); catch it by comparing the table's predicted maximum reach (≈ sum of a/d) to the arm's physical reach — a near-zero predicted reach flags the error. Partial: identifies the missing link lengths without the reach-comparison check.
