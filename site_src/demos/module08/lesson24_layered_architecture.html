Inject latency into one layer. The fast inner loop is latency-critical — delay there destabilises it. The slow outer layer (planning) tolerates delay easily. Layering by timing puts each job where its timing demands are met.
The inner loop (1 kHz control) must run on a real-time target with bounded latency; the outer loop (planning, perception, a few Hz) can live on a general-purpose host. Same robot, two timing worlds.
