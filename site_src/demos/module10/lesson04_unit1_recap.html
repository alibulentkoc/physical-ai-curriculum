The binding is what makes it a twin. Real robot and twin side by side, the four Module-9 inputs flowing through the sync arrow. A model is static; a simulation runs but floats free; only a live binding to the real asset earns the name "twin."
model — static description, no run
simulation — runs, but unbound
twin — runs and bound to reality
Unbind the sync and the twin degrades to a mere simulation — running, but no longer mirroring anything real.
