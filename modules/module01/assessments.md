## 4. Mastery thresholds & grading policy (FINAL · D-015)

**Mastery thresholds:**

| Level | Score | Meaning |
|---|---|---|
| Mastery | ≥ 85% | Strong readiness for Module 2 |
| Proficient | 70–84% | Ready, light review of weak units |
| Developing | 50–69% | Targeted remediation before Module 2 |
| Beginning | < 50% | Repeat key units |

**Knowledge Checks:** formative only — unlimited attempts, immediate feedback. Participation credit applies only where an LMS requires a score; they are not summatively graded.

**Competency-based exit (overrides numeric score):** A passing numeric grade alone is **not** sufficient to progress to Module 2. Students must demonstrate competency in all six areas:

1. Vector representation and operations
2. Coordinate frames
3. Matrix transformations
4. Trigonometric reasoning
5. Computational mathematics in Python
6. Workspace modeling

Competency overrides the numeric score in both directions: failing to demonstrate a competency blocks progression even with a passing percentage.
